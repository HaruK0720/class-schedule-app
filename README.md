# class-schedule-app（個人向け最適化・時間割管理アプリケーション）

学校で配布される複雑な時間割の情報を整理し、「今日、自分が本当に必要な情報」だけをスマホファーストで瞬時に確認できる、学生向けのUI/UXに特化した時間割管理アプリケーションです。

<br>

## 開発の背景と解決したい課題

### 【現状の課題】
学校から配布される従来の時間割表には、以下の不満点や課題がありました。
- **情報のノイズが多い**: 他クラスや他専攻の情報が1つの表に混在しており、自分に関係のある授業を判別するのに認知的なコストがかかる。
- **モバイル環境での視認性の低さ**: 文字が小さくレイアウトが複雑なため、通学時や移動中にスマートフォンから確認しづらい。
- **重要情報の分散**: 教室の場所、必要な持ち物、担当教員、オンライン授業のZoom情報（URL/ID/Pass）など、「付随する重要情報」をパッと一覧で確認する手段がない。

### 【本アプリによるアプローチ】
これらの課題を解決するため、 **「自分のクラスの情報だけを、圧倒的に見やすく」** をコンセプトに開発をスタートしました。
一週間の大まかな流れを俯瞰する「1週間の時間割表ビュー」と、当日のタスクや次の行動に集中するための「本日集中ビュー（持ち物・教室・リアルタイムの授業ハイライト）」をシームレスに切り替えられる、実用性と直感性を最優先した設計にしています。

<br>

## 主な機能

- **スマホファーストのカード型UI**: 画面幅の狭いデバイスでもストレスなく閲覧できる、親指1本で操作可能なデザイン。
- **動的な時間割ビュー（一週間の一覧）**: システムの現在時刻と連動し、**「今の時限・今日の曜日」を自動で強力にハイライト**する表。
- **本日集中ビュー（ローテーション機能）**: 今日の授業、教室、担当教員、必要な持ち物をリスト化。さらに、**終了した授業を自動でスキップし、次に受けるべき授業が常に最上部にくるローテーションロジック**を実装。
- **Zoom連携＆高度な入力管理** : オンライン授業のURL遷移に加え、ミーティングIDやパスコードを個別に管理・1タップコピー可能。授業タイプが「オンライン」の時のみ、専用の入力フォームが表示される動的UIを採用しています。
- **CRUD機能** : アプリケーション上から直接、授業の 「新規追加」「内容編集」「削除」 が可能です。

<br>

## 構成・技術選定

モダンなフロントエンド開発のベストプラクティスを網羅・学習するため、以下の技術スタックを採用しています。

### フロントエンド
- **React**: コンポーネント指向による高い保守性と再利用性を確保。Viteと組み合わせることで高速な開発を実現。
- **TypeScript**: インターフェースを用いた厳格な型定義により、実行前バグを徹底的に排除した堅牢なコードを記述。
- **styled-components**: コンポーネントの「状態（Props）」に応じた動的なスタイル制御（授業タイプごとの背景色分岐、リアルタイムハイライトなど）を実現。また、CSS変数をブレンドすることで**システムのダークモードへの自動シンクロ**にも対応。

### インフラ・バックエンド
- **Vercel**: GitHubリポジトリと連携したCI/CD（自動ビルド・デプロイ）環境を構築し、高速な配信を実現。
- **Supabase**: データベース化、ユーザーごとのカスタマイズ機能、認証機能（Auth）のクラウド拡張を見据えたBaaSの採用。

#### データベース設計（Supabase）

lessonsテーブルにて、授業情報を構造化して保存しています。

- ID管理: 曜日・時限・タイムスタンプ等を組み合わせた一意のキー設定。
- JSONB連携: zoom_info カラムに URL/ID/Passcode をオブジェクトとして格納し、柔軟なデータ操作を可能にしています。


#### セットアップと環境変数

ローカル開発およびデプロイ時には、以下の環境変数が必要です。

- VITE_SUPABASE_URL: SupabaseプロジェクトのURL
- VITE_SUPABASE_ANON_KEY: SupabaseのAnon Key

これらはセキュリティ保護のため、`.env` ファイルにて管理し、GitHubには公開しません。Vercel等の本番環境では、管理画面の Environment Variables から設定を行います。

### 開発環境・ツール
-  **Vite**: 非常に高速なビルド環境による開発効率の向上。
- **VSCode**: 拡張機能を活用したフロントエンド開発の最適化。
- **GitHub**: Gitによるバージョン管理およびリポジトリ管理。

## フォルダ構成と設計思想

本プロジェクトでは、アプリケーションの規模拡大（スケールアウト）およびSupabaseによるユーザー認証・データベース拡張を見据え、**「Page（ロジック統括）」「Component（UI部品）」「Style（装飾）」「Type（型定義）」「Lib（外部インフラ接続）」の5つの関心を100%分離するディレクトリ設計**を採用しています。

単一責任の原則を意識し、非常に高い保守性と見通しの良さを確保しています。

```text
src/
├── components/                 # 純UI部品の集約
│   ├── Auth.tsx                # 子：Supabase Authを利用したログイン/新規登録画面
│   ├── EditLessonModal.tsx     # 子：ユーザーごとの時間割情報を編集・登録するモーダル
│   ├── LessonCard.tsx          # 子：本日集中ビューの「授業カード」1コマ分の描画
│   ├── StatusHeader.tsx        # 子：現在の授業ステータスおよび通知バーの表示
│   └── TimetableMatrix.tsx     # 子：一週間の一覧ビュー（表）の描画
├── data/
│   └── timetable.json          # 時間割ソースデータ（※Supabase完全移行後は.gitignore対象）
├── lib/                        # 外部サービス（BaaS）との接続初期化設定
│   └── supabase.ts             # Supabaseクライアントの初期化（環境変数の読み込み）
├── pages/                      # 画面を配置
│   └── Timetable.tsx           # 親：認証セッションの監視、状態（State）管理、時間計算、各UI部品の統括
├── styles/                     # スタイル定義の集約ディレクトリ
│   └── timetable.styles.ts     # styled-componentsによるスタイル定義（Propsによる動的CSS）
├── types/                      # 型定義の集約ディレクトリ
│   └── timetable.ts            # Lesson型やProps型など、アプリ全体で共有するドメイン型
├── App.tsx                     # アプリケーションのルートコンポーネント（認証状態による画面分岐など）
├── index.css                   # グローバルスタイル・ダークモード用CSS変数（var--*）定義
└── main.tsx                    # アプリケーションのエントリーポイント
```
